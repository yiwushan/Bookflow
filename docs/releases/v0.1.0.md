# BookFlow v0.1.0 发布说明

发布时间：2026-03-30

## 版本目标
V0 重构版（章节原文 PDF 优先）：
1. 目录驱动切分。
2. 章节原文 PDF 阅读。
3. 双列 Feed + 互动进度 + 回忆推荐框架。

## 能力清单
1. PDF 导入支持目录来源优先级：`PDF Outline` > `manual_toc` > `pending_manual_toc`。
2. 目录叶子小节物化为章节 PDF，并持久化到：
   - `data/books/derived/<book_id>/<chunk_id>.pdf`
3. `POST /v1/toc/save` 升级为“保存并物化”，返回：
   - `materialized_chunks`
   - `generated_pdf_count`
   - `failed_entries`
   - `warnings`
4. `GET /v1/toc/pending` 增加状态字段：
   - `needs_manual_toc`
   - `toc_source`
   - `materialization_status`
   - `materialized_chunk_count`
5. `GET /v1/chunk_detail` 增加字段：
   - `content_type`
   - `section_pdf_url`
   - `page_start`
   - `page_end`
6. 新增 `GET /v1/chunk_pdf`，供 Reader 直接读取章节 PDF。
7. Feed 默认排序改为 V0 轻量混排：未完成优先 + 跨书交错 + 轻随机。
8. Book 拼图阅读完成口径收敛为 `section_complete`。

## 已知限制
1. 无目录 PDF 不做 OCR 自动识别，只进入 `pending_manual_toc` 队列。
2. 章节过长仅打 `long_section_warning`，V0 不自动再拆分。
3. `toc/save` 的“保存 annotation”与“DB 物化”不是单事务（文件系统 + DB 双介质）。
4. 章节 PDF 读取接口当前为整文件返回，不支持 HTTP Range 优化。

## Deferred 清单
1. OCR 自动目录识别与自动定位。
2. AI 重写/压缩。
3. 复杂语义推荐与向量召回。

## 验收口径
1. 导入有 Outline 的 PDF 后，Feed 能刷到章节卡，Reader 可直接读原文 PDF。
2. 导入无目录 PDF 后出现在 TOC pending；手工保存目录后立即可读。
3. `section_complete` 后 Book 拼图点亮并更新完成率。
4. smoke 与核心契约测试通过：
   - `./scripts/smoke_first_product.sh`
   - `python3 -m unittest tests.test_pdf_sectioning tests.test_api_contract`
